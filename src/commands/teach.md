# /teach

First-time setup for klamben. Run once when you add klamben to a
Flutter project. Gathers design context and saves it to `.klamben.md`
so all other commands give project-tailored advice.

## When to use

- First time using klamben in a project
- After a major rebrand or redesign
- When switching target platforms

## Process

### Step 1: Codebase scan (automatic)

Scan the project without asking questions. Read these files and
extract what you can:

1. **`pubspec.yaml`** — app name, Flutter/Dart SDK version,
   dependencies (`google_fonts`, `flutter_localizations`, `intl`,
   `flutter_svg`, `cached_network_image`, `flutter_platform_widgets`,
   `cupertino_icons`)
2. **`lib/` Dart files** — search for:
   - `ThemeData` — seed color, `useMaterial3`, `textTheme`, `colorScheme`
   - `ColorScheme.fromSeed` — extract seed color
   - `GoogleFonts.` — which font
   - `ThemeExtension` — spacing tokens, custom extensions
   - `Platform.isIOS` / `Platform.isAndroid` — platform branching
   - `.adaptive` constructors — `Switch.adaptive`, `Slider.adaptive`
   - `CupertinoPageRoute`, `CupertinoButton` — Cupertino usage
   - `Semantics`, `tooltip:` — accessibility patterns
   - `AppLocalizations`, `flutter_localizations` — i18n setup
   - `l10n.yaml` — localization config
3. **`analysis_options.yaml`** — existing lint rules

Build an internal summary of what you found. Note which questions
from Step 2 are already answered by the scan.

### Step 2: Ask questions (one at a time)

Skip any question already answered by the codebase scan. Use
multiple choice when possible.

**Q1: Target platforms** (skip if scan found platform branching)
> Which platforms does this app target?
- Android + iOS with platform-adaptive UI (Recommended)
- Android + iOS with Material-only UI
- Android only
- iOS only

**Q2: Brand personality** (always ask)
> Pick 2-3 words that describe this app's personality:
- Professional / Playful / Minimal / Bold / Warm / Technical /
  Premium / Friendly
- (or let the user type their own)

**Q3: Primary audience** (always ask)
> Who uses this app?
- General consumers
- Business/enterprise users
- Developers/technical users
- Kids/education
- (or open-ended)

**Q4: Accessibility tier** (skip if scan found Semantics usage)
> What level of accessibility do you target?
- WCAG AA (Recommended)
- WCAG AAA (strict)
- Best-effort (no formal target)

**Q5: Design maturity**
> What's the state of this project's design?
- Greenfield — no design system yet
- Has brand colors + fonts but no formal tokens
- Has a design system (Figma/tokens/components)
- Following an external design spec

**Q6: Seed color** (skip if `ColorScheme.fromSeed` found in scan)
> What's your primary brand color? (hex, e.g. #6750A4)
- Let the user type a hex value
- Default suggestion: `#6750A4` (Material 3 default)

### Step 3: Write `.klamben.md`

Write the following file to the project root. Fill in values from
the scan and user answers. Use ✓ for features present and ✗ for
features missing.

```markdown
# klamben Design Context

> Generated by `/teach`. Edit freely — klamben commands reference
> this file for project-specific guidance.

## Project

- **App:** [app name from pubspec.yaml]
- **Platforms:** [answer from Q1]
- **SDK:** Flutter [version] / Dart [version]

## Brand

- **Personality:** [answer from Q2]
- **Audience:** [answer from Q3]
- **Seed color:** [answer from Q6 or scanned value]

## Design Decisions

- **Theme:** [scanned ThemeData details] ✓/✗
- **Font:** [scanned font or "not configured"] ✓/✗
- **Spacing tokens:** [scanned ThemeExtension or "not set up"] ✓/✗
- **Dark mode:** [scanned brightness setup] ✓/✗
- **Platform adaptation:** [scanned adaptive patterns] ✓/✗
- **Localization:** [scanned l10n setup] ✓/✗
- **Accessibility target:** [answer from Q4]

## Design Maturity

[answer from Q5 + any relevant scan findings]

## Existing Patterns Found

[List specific files and patterns discovered during scan, e.g.:]
- ThemeData at `lib/theme/app_theme.dart`
- Spacing tokens at `lib/theme/spacing.dart`
- No Semantics widgets found — run `/check-a11y`
- No i18n setup — run `/localize`

## Recommendations

Based on the scan, consider running:
- [list relevant commands based on gaps found]
```

After writing, tell the user:
> "Design context saved to `.klamben.md`. All klamben commands will
> now reference this file for project-specific guidance. You can edit
> it anytime. Run `/audit` to see how your project scores."

## If `.klamben.md` already exists

Ask the user:
> ".klamben.md already exists. Want to re-run teach (overwrites
> existing context) or keep the current one?"

If they choose to keep it, exit without changes.

## Do NOT

- Make code changes — this is a setup command only
- Skip the scan — always scan first, then ask
- Ask questions the scan already answered
- Write to any file other than `.klamben.md`
