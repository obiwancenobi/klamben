# HTML Report Output for klamben CLI

## Context

klamben CLI currently outputs findings as plain text or JSON. Users want a polished, shareable HTML report with modern UI, charts, and a health dashboard. This adds two capabilities:

1. `--format=html` on `detect` command — same findings, rendered as premium HTML
2. New `report` command — richer health dashboard with score, category breakdown, recommendations

## Decisions

- **Visual direction:** Gradient Premium — dark base (#0f172a), purple-to-cyan gradient accents, glass-morphism cards
- **File handling:** Default save to `klamben-report.html`, optional `--output` override, print path to console, no auto-open
- **Self-contained:** Single HTML file, zero external deps. CSS-only charts, inline SVG where needed, vanilla JS
- **Architecture:** HTML template as embedded Dart const. Dart injects data as JSON. Template JS handles rendering
- **History:** v1 is single-run. Data structure designed to accept history array later without breaking changes

## Architecture

```
detect_command.dart
  ├── --format=text → text_reporter.dart (existing)
  ├── --format=json → json_reporter.dart (existing)
  └── --format=html → html_reporter.dart (NEW)
                          ├── Builds ReportData from List<Finding>
                          ├── Serializes to JSON
                          ├── Injects into HTML template
                          └── Writes to output path

report_command.dart (NEW)
  ├── Runs engine (same as detect)
  ├── Builds enriched ReportData (health score, coverage, recommendations)
  └── Uses html_reporter with dashboard template
```

## Data Model

```dart
class ReportData {
  final DateTime generatedAt;
  final String scannedPath;
  final int totalFiles;
  final int totalFindings;
  final Map<String, int> bySeverity;      // {error: 5, warning: 12, info: 8}
  final Map<String, int> byCategory;      // {visual: 7, layout: 6, ...}
  final Map<String, List<Finding>> byFile; // grouped findings
  final List<RuleHit> topRules;           // most triggered rules, sorted desc
  final int healthScore;                   // 0-100
  final int totalRules;                    // 24
  final int triggeredRules;               // distinct rules that fired

  Map<String, dynamic> toJson();          // for template injection
}

class RuleHit {
  final String ruleId;
  final String title;
  final RuleSeverity severity;
  final int count;
}
```

**Health score formula:** `max(0, 100 - (errors * 10 + warnings * 3 + infos * 1))`

## Template Injection

```html
<script>
  const KLAMBEN_DATA = /* INJECT_DATA */;
</script>
```

Dart replaces `/* INJECT_DATA */` marker with JSON-serialized ReportData. Template JS reads `KLAMBEN_DATA` and populates DOM on load.

## HTML Template Sections

### Detect Report (`--format=html`)

1. **Header** — klamben name + "Design Report", scan path, timestamp, file count
2. **Health Score** — large conic-gradient donut (0-100), centered
3. **Summary Cards** — 4 glass-morphism cards: Errors (red glow), Warnings (amber), Info (blue), Total Files
4. **Severity Bar** — horizontal stacked bar showing error/warning/info proportions
5. **Category Breakdown** — 4 horizontal bars (visual, layout, platform, code-quality) with labels + counts
6. **Top Rules** — ranked list, bar width proportional to hit count, severity badge
7. **Findings Detail** — collapsible file groups, each finding shows line:col + severity pill + rule-id + message
8. **Footer** — "Generated by klamben" + version + timestamp

### Dashboard Report (`report` command) — extends detect template with:

9. **Rule Coverage Gauge** — "18/24 rules triggered" as progress bar
10. **Category Health** — per-category score as 4 progress bars
11. **Recommendations** — top 3 highest-impact rules to address, with fix hints from RuleMetadata

## Visual Design Specs

- **Background:** `#0f172a` (slate-900)
- **Card background:** `rgba(255,255,255,0.05)` with `backdrop-filter: blur(12px)`, border `rgba(255,255,255,0.1)`
- **Gradient accents:** `linear-gradient(135deg, #a78bfa, #38bdf8)` (purple to cyan)
- **Severity colors:** error `#f87171`, warning `#fbbf24`, info `#38bdf8`
- **Category colors:** visual `#a78bfa`, layout `#38bdf8`, platform `#34d399`, code-quality `#fb923c`
- **Typography:** system-ui stack, health score 48px bold with gradient text
- **Charts:** CSS-only (conic-gradient for donut, flex-based bars, CSS transitions on load)
- **Hover effects:** subtle glow + slight scale on cards

## CLI Interface

```bash
# Detect with HTML output (default path)
klamben detect --format=html lib/
# Output: Report written to klamben-report.html

# Detect with custom output path
klamben detect --format=html --output=reports/scan.html lib/

# Dashboard report (always HTML)
klamben report lib/
# Output: Report written to klamben-report.html

klamben report --output=dashboard.html lib/
```

**Exit codes:** Same as detect (0 = no findings, 1 = findings found, 2 = error).
